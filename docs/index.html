Benchmark method

The quality score uses one model slot and one benchmark worker. It is separate from saturated 12-slot throughput testing.

Serving262,144 total context, one fixed slot, unified KV, F16 KV, flash attention.
Benchmark69 agent scenarios, one benchmark worker, 180-second timeout, temperature 0, thinking disabled.
ControlPinned model checksum, pinned runtime, pinned benchmark commit, seed 42.
llama-server -m Qwen3.6-35B-A3B-UD-Q5_K_XL.gguf -ngl 99 -ncmoe 0 \
  --no-op-offload -c 262144 -np 1 -kvu -fa on -ctk f16 -ctv f16 \
  -b 8192 -ub 1024 -t 15 --jinja --metrics

tool-eval-bench --backend llamacpp --no-think --seed 42 \
  --reference-date 2026-03-20 --parallel 1 --timeout 180
NOTE Separate 12-slot measurements characterize serving throughput under concurrency; they are not part of this quality score.

How the experts route

A source-grounded map of Qwen's top-8 MoE decision and Treebeard's Intel and NVIDIA execution paths.

Qwen chooses eight of 256 routed experts per token and adds one separately gated shared expert. Treebeard preserves that decision, then schedules the selected work for Intel SYCL or NVIDIA CUDA.
